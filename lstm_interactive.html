LSTM — Long Short-Term Memory
Forget Gate · Input Gate · Cell State Highway · Output Gate · Why It Solved Vanishing Gradients
01
The Problem
Vanilla RNN fails on long sequences
Gradient multiplied by Wₕ at each step
If ‖Wₕ‖ < 1: gradient → 0 (vanish)
∂L/∂h₀ = Πₜ ∂hₜ/∂hₜ₋₁ → 0
✅
LSTM (1997)
Hochreiter &
Schmidhuber
solved this
02
Two State Vectors
LSTM adds a cell state cₜ
hₜ — hidden state (short-term memory)
cₜ — cell state (long-term memory)
Both flow timestep to timestep
✅
cₜ is the key
innovation —
gradients flow
almost unchanged
03
Forget Gate
What % of old memory to keep?
σ → outputs 0 (erase) to 1 (keep)
Applied elementwise to cₜ₋₁
f = σ(W_f · [hₜ₋₁, xₜ] + b_f)
✅
Example: reading
new subject →
forget old topic
f ≈ 0 for old
04
Input Gate
What new info to write to cell?
Two parts: gate i (how much) + g̃ (what)
Combined: i ⊙ g̃ added to cell
i = σ(W_i·[hₜ₋₁,xₜ]+b_i) g̃ = tanh(...)
✅
i gates what
gets written
g̃ is the new
candidate value
05
Cell State Update
The highway for gradients
Old cell × forget + new candidate × input
Gradient flows back via addition — no vanish!
cₜ = f⊙cₜ₋₁ + i⊙g̃
✅
Addition not
multiplication →
gradient stays
alive 1000s steps
06
Output Gate
What to expose from cell?
Not all of cₜ is useful right now
Output gate filters what h sees
o = σ(W_o·[hₜ₋₁,xₜ]+b_o) hₜ = o⊙tanh(cₜ)
✅
tanh squashes
cₜ to [-1,1]
o gates how
much to expose
